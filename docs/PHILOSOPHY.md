# 不断捨離 (Indecision Declutter) - プロダクト哲学 & デザイン

## コアコンセプト: 「一時退避ツール」

### 課題: 「迷い」が片付けを止める
片付けが失敗するのは、私たちが怠惰だからではなく、**手が止まってしまうから**です。
「これ、捨てようかな、どうしようかな？」と迷った瞬間、物理的なアクションが停止し、そこから精神的な疲れ（決断疲れ）が始まります。
既存のアプリやメソッドは「捨てる」か「残す」かの二択をその場で迫るため、このプレッシャーがさらに手を止めさせます。

### 解決策: 「決断しないで、退避する」
このアプリは、片付けのフローから**決断を取り除く**ために設計されています。
その場で白黒つけるのではなく、「シェルター（一時退避所）」を提供します。

- **捨てるか迷ったら撮る**
- 写真に撮ってデジタル化することで、物理的なモノと「決断プロセス」を切り離します。
- 「記憶（写真）は確保された」という安心感とともに、物理的なモノを手放す（あるいは保留ボックスに入れる）ことができます。

---

## 哲学: 記憶の連続性 (Continuity of Memory)

私たちがモノを捨てるのを恐れるのは、**忘却**を恐れるからです。
そのモノが物理的に消えてしなえば、それに紐づく思い出も消えてしまうような気がするのです。

**不断捨離は「記憶の連続性」を保証します。**
- たとえ物理的なモノが手元からなくなっても、その「デジタル・ツイン」は残り続けます。
- このアプリにおける「手放す（断捨離完了）」というアクションは、削除ではありません。
- それは「散らかったモノ」から「アーカイブ（捨離リスト）」への**変容（Transformation）**です。
- この安心感こそが、モノへの執着を手放す鍵となります。

## デザイン美学: 「禅と現代」 (Zen & Modern)

UIは意図的に、静かで、ミニマルで、モノに対して敬意を払うデザインになっています。
焦らせるような通知や、派手な装飾は一切排除しています。

### コンセプト: 墨と和紙 (Sumi & Washi)

日本の伝統的な書画の美しさを、現代のデジタルUIに落とし込みました。
真っ黒（`#000`）や真っ白（`#FFF`）を使わず、目に優しく深みのある色を採用しています。

#### カラーパレット

| 色名 | Hex Code | 用途 | 意味 |
| :--- | :--- | :--- | :--- |
| **Sumie (墨絵)** | `#1c1c1e` | テキスト、アクセント | 深い濃いグレー。完全な黒よりも温かみがある。 |
| **Washi (和紙)** | `#f9f9f7` | 背景 | わずかに黄みがかったオフホワイト。紙の質感。 |
| **Shu (朱)** | `#ef4444` | 今すぐ (Urgent) | 印鑑のような鮮やかな赤。行動を促す。 |
| **Yamabuki (山吹)** | `#f59e0b` | 捨てたい (Discard) | 実りのある決断を表すオレンジ。 |
| **Kitsune (狐)** | `#eab308` | 迷い (Hesitate) | 曖昧さを許容する黄色。 |
| **Matsuba (松葉)** | `#10b981` | 残す (Keep) | 永続性を象徴する深い緑。 |

### インタラクション: 静かなる支援 (Silent Support)

- **ノイズを消す:** 撮影モードに入るとUIは最小限になり、目の前のモノと対話する空間を作ります。
- **罪悪感を与えない:** 「迷い」や「残す」ことを責めません。それもまた一つの大切な状態として扱います。
- **手触り:** ボタンやカードは、押した時にわずかに沈み込むなどのフィードバック返し、確かな操作感（手触り）を提供します。

## 技術的選択: PWA & Local First

なぜネイティブアプリではなく、クラウドを使わないPWAなのか？

- **プライバシー:** 片付けは極めて個人的な行為です。散らかった部屋の写真をクラウドにアップロードしたくないユーザーのために、データは全て端末内（IndexedDB）に閉じます。
- **永続性:** サーバー代が払えなくなってサービスが終了する…ということはありません。アプリは手元に残り続けます。
- **速度:** 「迷い」よりも早く動く必要があります。ネットワーク遅延ゼロのローカル動作が最適です。
