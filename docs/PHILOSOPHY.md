# 不断捨離 (Indecision Declutter) - プロダクト哲学 & デザイン言語

> このドキュメントは、コンセプト・世界観・デザイン言語（トンマナ／カラー／タイポ／円相）の**正典**です。
> 画面別の詳細仕様は `docs/specs/product-spec.md`、カラーの実値は `src/theme/tokens.ts`、円相の描画要件は `openspec/specs/enso-motif/spec.md` を参照してください。

## コアコンセプト: 「モノの縁側」

### 課題: 「迷い」が片付けを止める
片付けが失敗するのは、私たちが怠惰だからではなく、**手が止まってしまうから**です。
「これ、捨てようかな、どうしようかな？」と迷った瞬間、物理的なアクションが停止し、そこから精神的な疲れ（決断疲れ）が始まります。
既存のアプリやメソッドは「捨てる」か「残す」かの二択をその場で迫るため、このプレッシャーがさらに手を止めさせます。

### 解決策: 「決めずに、写しを収める。」
このアプリは、片付けのフローから**決断を取り除く**ために設計されています。
その場で白黒つけるのではなく、捨てるでも残すでもない「モノの縁側」を提供します。

- **捨てるか迷ったら撮る**
- 写真に撮って写しを収めることで、物理的なモノと「決断プロセス」を切り離します。
- 「記憶（写真）は確保された」という安心感とともに、物理的なモノを手放す（あるいは保留ボックスに入れる）ことができます。

## 哲学: 記憶の連続性 (Continuity of Memory)

私たちがモノを捨てるのを恐れるのは、**忘却**を恐れるからです。
そのモノが物理的に消えてしまえば、それに紐づく思い出も消えてしまうような気がするのです。

**不断捨離は「記憶の連続性」を保証します。**
- たとえ物理的なモノが手元からなくなっても、その「デジタル・ツイン」は残り続けます。
- このアプリにおける「手放す（断捨離完了）」というアクションは、削除ではありません。
- それは「散らかったモノ」から「アーカイブ（捨離リスト）」への**変容（Transformation）**です。
- この安心感こそが、モノへの執着を手放す鍵となります。

## デザイン美学: 「禅と現代」 (Zen & Modern)

UIは意図的に、静かで、ミニマルで、モノに対して敬意を払うデザインになっています。
焦らせるような通知や、派手な装飾は一切排除しています。

### コンセプト: 墨と和紙 (Sumi & Washi)

日本の伝統的な書画の美しさを、現代のデジタルUIに落とし込みました。
真っ黒（`#000`）や真っ白（`#FFF`）を使わず、目に優しく深みのある色を採用しています。

## 円相モチーフ (Enso)

円相はアプリの核です。**色が状態を語り**、閉じ具合が思想を語ります。
（描画・アニメの要件正典は `openspec/specs/enso-motif/spec.md`。実装は `src/components/EnsoMark.tsx` / `EnsoImage.tsx`。）

- **青（勝色）＝開いた円相＝縁側** … 写しは収まったが、捨てる／残すの判断は開いたまま。モノは縁側にいる。
- **鈍色＝グレーアウトの円相＝手放し確認** … 青の円相を沈めた弔いの静けさ。まだ手放していない、静かな決心の場。
- **朱＝閉じた円相＝手放し・捨離** … 変容・完結。閉じて結ばれ、写真は灰へと薄れる。

**描画の原則:**
- 常に**フル描画**の筆円。細い幾何学リングには決して退化させない。
- リビールは**一筆書き**（左下から一続き、約 420–520ms の落ち着いた運び）で円を現す。
- 筆の**かすれ（渇筆）・濃淡・テクスチャを保持**する。均一幅のベクターストロークへの自動トレース置換は禁止（本物の筆 PNG テクスチャを使う）。
- 円は**閉じきらなくてよい**＝優柔不断の肯定。

## コピー／トンマナ原則 (Voice & Tone)

言葉づかいが世界観を保ちます。実装時の判断基準:

- モノを記録する行為は **「写しを収める」**。
- 未決定のモノの居場所は **「縁側」**（内部識別子は `candidate`）。
- 手放しは **「削除ではなく変容」**。移り先は **「捨離」**（内部識別子は `discarded`）、`releasedAt` を刻む。
- 「残す」は縁側の中の度合いであって、別ステータスではない。
- 罪悪感を与えない。「迷い」も「残す」も、責めずに一つの大切な状態として扱う。
- **静けさ・余白・和紙の温かみ**を最優先。装飾より「間」。
- ノイズを消す。撮影に入ると UI は最小限になり、目の前のモノと対話する空間を作る。
- 手触り。ボタンやカードは押すとわずかに沈み、確かな操作感を返す。

## 技術的選択: ローカルファースト (Local First)

現行スタックは **Expo + React Native**。データは端末内に閉じ、クラウドに依存しません。

- メタデータ（品名・度合い・記録など）は端末内の **SQLite**（`expo-sqlite` / `src/lib/db.ts`）。
- 写真は端末内の **ファイルシステム**（`expo-file-system` / `src/lib/photoStorage.ts`、`indecision-photos` ディレクトリ）に保存。

なぜクラウドを使わないのか:

- **プライバシー:** 片付けは極めて個人的な行為です。散らかった部屋の写真をクラウドにアップロードしたくないユーザーのために、データは全て端末内に閉じます。
- **永続性:** サーバー代が払えなくなってサービスが終了する…ということはありません。アプリは手元に残り続けます。
- **速度:** 「迷い」よりも早く動く必要があります。ネットワーク遅延ゼロのローカル動作が最適です。
