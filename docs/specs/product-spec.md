# 不断捨離 プロダクト仕様

状態: draft
正本: `docs/design/design_handoff_indecision_declutter` と `AGENTS.md`
対象: ルートプロジェクトの React Native / Expo アプリ

## 1. プロダクト原則

不断捨離は、次の一点を中心に設計する。

> 決断しないで、退避する。

このアプリの役割は、片付け中に「捨てるか残すか」を決められず手が止まる瞬間を、写真と記録の一時退避で受け止めること。主要アクションは保存・削除・整理ではなく **退避**。写真で記憶を確保し、物理的な片付けを止めない。

手放しは削除ではない。写真と記録を残したまま、候補から捨離へ移す **変容** として扱う。

## 2. スコープ

製品アプリはルートプロジェクトの React Native / Expo で実装する。`legacy/vue-pwa/` は参照用であり、実装先ではない。

アプリはローカルファーストとする。写真と記録は端末内に保存し、明示依頼がない限りバックエンド、アカウント、同期、クラウド保存を追加しない。

`docs/design/design_handoff_indecision_declutter` 配下の `.dc.html` は見た目・コピー・導線の参照プロトタイプであり、HTML を製品コードへコピーしない。

## 3. 用語とコピー

- 撮影して保持する行為は **退避** と呼ぶ。ユーザー向けコピーでは原則として「保存」を使わない。
- 候補から捨離へ移す行為は **手放す** と呼ぶ。
- 手放しは削除ではなく **変容** として扱う。
- 「写真と記録は捨離に残る」ことを安心材料として明示する。
- 撮影後の既定の捨てたい度は **迷い**。

## 4. ビジュアル言語

共有する色・フォント・角丸・影などの定数は `src/theme/tokens.ts` に集約する。

必須モチーフ:

- 和紙背景: 温かい紙面。
- Shippori Mincho: アプリ名、品名、儀式コピー、想いメモ。
- Inter: UI ラベル、ボタン、メタ情報。
- 青 / 開いた円相: 退避、未決定、保留。
- グレー / 鈍色円相: 手放し確認の静けさ。
- 朱 / 閉じた円相: 手放し完了、捨離、変容の完結。
- 円相は PNG アセットを使う。元ベクターがない限り、自動トレース SVG へ置き換えない。

主要カラー:

- 和紙: `#f4f1ea`
- シート: `#efece3`
- カード / ノート: `#ffffff`, `#fbfaf6`, `#faf8f3`
- 墨: `#2a2a2c`
- 勝色: `#2c2e63`
- 朱: `#9e3b2c`
- 鈍色: `#57565b`

捨てたい度:

| 値 | 表示 | 意味 | 色 |
|---:|---|---|---|
| 3 | 今すぐ | すぐ手放したい | `#dc2626` |
| 2 | 捨てたい | 手放したい | `#ea580c` |
| 1 | 迷い | 未決定。既定値 | `#ca8a04` |
| 0 | 残す | 残す方向 | `#16a34a` |

捨てたい度は候補内の度合いであり、ステータスではない。捨離への移動は別アクション。

## 5. データモデル

標準の item 型:

```ts
type ItemStatus = 'candidate' | 'discarded';
type UrgencyLevel = 0 | 1 | 2 | 3;

interface Item {
  id: string;
  name: string;
  photos: string[];
  coverIndex: number;
  urgency: UrgencyLevel;
  status: ItemStatus;
  blockers: string[];
  memoryNote?: string;
  price?: number;
  lastUsedAt?: string;
  location?: string;
  createdAt: number;
  updatedAt: number;
  releasedAt?: number;
}
```

永続化要件:

- item と写真は端末内に保存する。
- 候補一覧は `status === 'candidate'` を表示する。
- 捨離一覧は `status === 'discarded'` を表示する。
- 手放し時は `status = 'discarded'` とし、`releasedAt = Date.now()` を記録する。
- 候補に戻すときは `status = 'candidate'` とし、`releasedAt` を消す。
- 写真順は安定して保持する。表紙は `coverIndex` で表す。
- 写真削除・並べ替えで `coverIndex` が壊れないようにする。
- 通常起動でサンプルの私物データを自動投入しない。デモデータは開発・デモモードに限定する。

## 6. ナビゲーション

React Navigation を使う。

- Bottom tabs: `CandidateList`, `DiscardedList`
- 中央カメラ FAB: `Capture` を開く
- Stack push: 候補詳細、編集、捨離詳細
- Modal: 写真ビューア、手放し確認
- 完了演出: 手動導線を持つ全画面ページ

必要な画面:

| 画面 | 役割 |
|---|---|
| Candidate tab | 候補ワークベンチ |
| Discarded tab | 捨離記録 |
| Capture | 撮影、レビュー、捨てたい度選択 |
| EvacuationComplete | 青い円相の退避完了 |
| ItemDetail | 候補詳細 |
| ItemEdit | 記録編集 |
| PhotoViewer | 全画面写真ビューア |
| ReleaseConfirm | 手放し確認モーダル |
| ReleaseComplete | 朱の円相の手放し完了 |
| DiscardedDetail | 捨離詳細 |

## 7. 画面仕様

### 01. 候補一覧

目的: 物理的な片付けから一時退避したモノを一覧する。

必須要素:

- ヘッダー: `不断捨離` と設定ボタン。設定項目が未定の間は準備中の案内を表示する。
- 横スクロールの捨てたい度フィルタ: 全部、今すぐ、捨てたい、迷い、残す。
- 既定の2列カードグリッドと、切り替え可能なコンパクト縦リスト。
- 候補タブが active のボトムナビと中央カメラ FAB。
- 初回用のカメラ吹き出し: `捨てるか迷ったら、撮る`。

挙動:

- カード tap で詳細へ遷移する。
- FAB tap で撮影へ遷移する。
- 捨てたい度は実際に一覧を絞り込む。迷った理由は記録として保持するが、一覧の絞り込みには使わない。

### 02. 撮影

目的: 1つのモノに対して複数写真を撮り、捨てたい度を選んで退避する。

必須要素:

- アプリ内カメラのプレビュー領域（FAB から中間タップなしで即起動）。
- ズームコントロール: ピンチズームと `1x` / `2x` プリセット、前面/背面カメラ切替。
- ライブラリからの写真選択導線。
- `やめる` と `撮り直す`。
- 表紙表示つき写真ロールと `もう1枚`（カメラ/ライブラリの選択）。
- 写真枚数、捨てたい度 4択、2つの退避アクションを持つ和紙シート。
- 主ボタン: `退避して、次を撮る`
- 副ボタン: `退避して、書き留める`

挙動:

- カメラ FAB から撮影に入ると即カメラが起動する。初回撮影を X でやめた場合は候補一覧へ戻る。
- 退避アクションは写真が 1 枚以上あるときだけ表示する。写真ゼロでは退避できない（まず撮影へ誘導）。
- ピンチでズームでき、値はカメラの許容範囲にクランプする。前面/背面切替時はズームを 1x に戻す。
- 既定の捨てたい度は `迷い`。捨てたい度アイコンの円相は選択肢の枠内に収める。
- `もう1枚` は同じ作業中 item に写真を追加する。カメラ撮影と端末ライブラリからの選択のどちらも選べる。
- `撮り直す` は現在の撮影分を取り直す。既存の保存済み記録には影響しない。
- `やめる` は item を作らず一覧へ戻る。
- `退避して、次を撮る` は item を作成し、退避完了を表示する。退避完了から `次を撮る` で撮影へ戻る。
- `退避して、書き留める` は item を作成し、編集へ遷移する。

### 03. 退避完了

目的: 退避できた安心感を出しつつ、片付けのリズムを止めない。

必須要素:

- 和紙背景。
- 青い開いた円相と、その中の写真。
- 緑のチェックバッジ。
- `退避しました`。
- 実データに基づく `本日 N つめ`。
- `退避を取り消す` スナックバー。

挙動:

- 自動遷移はしない。ユーザーが余韻を見届けてから `次を撮る` または `候補一覧へ戻る` を選ぶ。
- 取り消しは直前に作成した退避を無効化し、元の撮影 / 一覧文脈へ戻る。
- 青い円相は閉じきらない。未決定を肯定する見た目にする。

### 04. 候補エンプティ

目的: 初回利用時もマーケティング画面ではなく、使えるアプリシェルとして見せる。

必須要素:

- 候補一覧と同じヘッダー、ボトムナビ、FAB。
- 中央の薄い青円相。
- `捨てるか迷ったら、撮る。`
- `迷いは、捨てなくていい。ひとまず、ここへ退避しましょう。`
- カメラ FAB へのヒント。

### 05. 候補詳細

目的: 候補 item のホーム。写真と記録を確認し、編集や手放しへ進む。

必須要素:

- 全画面の写真ヘッダー / カルーセル。
- 戻るボタン。
- 写真カウンタとドット。
- `写真を足す`。
- 主役としての品名。
- メタ情報: 捨てたい度、写真枚数、作成日。
- 円相区切り。
- 記録カード: 編集リンク、購入価格、保管場所、最後に使った日、迷った理由、想いメモ抜粋。
- 主ボタン: `手放す`。

挙動:

- 写真 tap で該当 index のビューアを開く。
- カルーセルは横移動できる。
- `写真を足す` は写真を追加する。カメラ撮影と端末ライブラリからの選択のどちらも選べる。
- 品名は tap でその場編集できる（編集画面へ遷移せずに保存）。
- `編集` で編集へ遷移する。
- `手放す` で手放し確認を開く。
- 未実装の破壊的操作を active UI として表示しない。

### 06. 編集

目的: item の記録をすべて編集する。

必須要素:

- ヘッダー: `キャンセル`, `記録する`, `完了`。
- 表紙バッジつき写真ロール。
- 写真追加、削除、表紙設定、並べ替え。
- 品名入力。
- 捨てたい度セグメント。
- 購入価格、最後に使った日、保管場所の入力。
- 迷った理由 / blocker。
- 想いメモの複数行入力。

挙動:

- 写真追加はカメラ撮影と端末ライブラリからの選択のどちらも選べる。
- 「最後に使った日」はカレンダー/デートピッカーで選ぶ。`今日` のクイック入力は残す。値は `YYYY-MM-DD` で保存する。
- `完了` で変更を永続化し、詳細へ戻る。
- ユーザー向けコピーでは「保存」を避ける。
- design handoff は自動保存前提。明示コミット方式を採るなら、仕様として意図的に差分化するか、自動保存へ寄せる。

### 07. 写真ビューア

目的: 写真を全画面で確認する。

必須要素:

- 黒背景。
- 閉じるボタン。
- 現在 index カウンタ。
- 表紙設定 / 削除の more メニュー。
- 大きな現在写真と左右のぞき。
- サムネイル帯。

挙動:

- 横スワイプで写真を切り替える。
- サムネイル tap で該当写真へジャンプする。
- more メニュー操作は item 状態へ反映する。
- 手放し済みの item では more メニューに `画像をライブラリに保存` を追加表示する（候補では出さない）。

### 08. 手放し確認

目的: confirm ダイアログではなく、手放しを静かな儀式として扱う。

必須要素:

- 全画面モーダル。
- グレー / 鈍色のトーン。
- グレー円相と暗い写真。
- `{name} を手放しますか？`
- 削除ではなく変容であることを伝える本文。
- 主ボタン: `手放す（捨離リストへ）`
- 副操作: `まだ、そばに置く`

挙動:

- 手放し実行時に `releasedAt` を記録し、item を捨離へ移す。
- キャンセルは状態変更せず詳細へ戻る。

### 09. 手放し完了

目的: 手放しが完結したことを前向きに示す。

必須要素:

- 朱の閉じた円相。
- グレーへ薄れる写真。
- `円が、ひとつ結ばれました`。
- `{name} を手放しました。記録は「捨離」に残ります。`
- `— 捨離リストへ移りました —`

挙動:

- 自動遷移はしない。ユーザーが余韻を見届けてから `捨離リストを見る` で捨離一覧へ移るか、`候補一覧へ戻る` で候補一覧へ戻る（退避完了と対称の導線）。
- この画面はデータへの副作用を持たない。
- 円相は閉じて完結した印象にする。

### 10. 捨離一覧

目的: 手放し済み記録を静かに振り返る。

必須要素:

- 共通ヘッダーと、捨離 active のボトムナビ。
- 区切り見出し: `手放した記録`。
- 候補一覧と同じ 2 列カードと、切り替え可能なコンパクト縦リスト。
- グレー / 減光された写真。
- `手放し済み` バッジまたは帯。
- 捨離専用エンプティ。

挙動:

- カード tap で捨離詳細へ遷移する。

### 11. 捨離詳細

目的: 変容済みの記録を読み返し、戻せる安心を残す。

必須要素:

- 控えめに暗い full-page 写真ヒーロー。
- 写真ヒーロー右上に、表示中の画像をライブラリに保存する `download`。
- 朱の閉じた円相バッジと `手放し済み`。
- 品名。
- 手放した日と `N日間そばに`。
- 朱の円相区切り。
- 読み取り専用記録カード（品名のみ例外的に編集可）。
- `やっぱり候補に戻す`。
- `手放したことにしたけれど、まだ手元にある時に`。

挙動:

- 品名は tap でその場編集できる。品名以外の記録は読み取り専用のまま。編集しても `status` と `releasedAt` は変わらない。
- `download` で表示中の写真を端末のフォトライブラリに保存する（Web はブラウザのダウンロード）。
- 候補に戻すと `status` を candidate に戻し、`releasedAt` を消す。

## 8. 操作品質

- 候補 / 捨離一覧、詳細、編集、捨離詳細は 375x812 の iPhone 相当カンプで破綻なく収まること。
- カードは安定した 2 列グリッドを維持すること。
- 完了画面は generic alert ではなく全画面の儀式として扱うこと。
- 見えているボタンや affordance は動作すること。未実装なら active UI として出さない。
- 候補 / 捨離の両タブと詳細 lookup に loading / empty / error 状態を持たせること。
