# ハンドオフ: 不断捨離（Indecision Declutter）UI 一新

「捨てられない」を「一時退避」して断捨離を止めないためのアプリ。優柔不断で片付けが苦手な人・思い出の品を捨てられない人向け。コアは **「決断しないで、退避する」**。世界観は禅／和モダン（墨と和紙）。本ハンドオフは、全11画面の高忠実カンプと全体フロー図を、既存の Vue コードベースに実装するための仕様書です。

---

## 0. 同梱ファイル

| ファイル | 内容 |
|---|---|
| `不断捨離_完成カンプ.dc.html` | **全11画面の高忠実モックアップ**（本仕様の視覚的な正） |
| `不断捨離_全体フロー図.dc.html` | 画面遷移を1枚で俯瞰する縦フロー図 |
| `assets/pwa-512x512.png` | 円相（勝色＝青）のロゴ画像。退避・アイコンに使用 |
| `assets/enso-shu-512.png` | 円相（朱）の画像。手放し完了に使用 |
| `assets/pwa-192x192.png` | PWA アイコン（小） |
| `support.js` | `.dc.html` をブラウザで開くためのランタイム（実装には不要） |

`.dc.html` はブラウザでそのまま開けます（同じ階層に `assets/` と `support.js` が必要）。

### これらは「デザインの参照」です
同梱の HTML は **意図した見た目と挙動を示すプロトタイプ**であり、そのまま製品コードにコピーするものではありません。タスクは、この HTML デザインを **既存の Vue コードベースの流儀（Vue 3 SFC / vue-router / Dexie / lucide-vue-next）で作り直す**ことです。

### 忠実度: 高忠実（hi-fi）
色・タイポグラフィ・余白・角丸・アイコン・コピーはすべて確定値です。ピクセル単位で再現してください。写真部分のみプレースホルダ（斜線）です。

## 1. 既存コードベースとの対応・差分

スタック: **Vue 3 + Vite + TypeScript + vue-router + Dexie(IndexedDB) + lucide-vue-next + vite-plugin-pwa**

### 現状
- ページ: `src/pages/WorkbenchPage.vue`（`/items`）, `DetailsPage.vue`（`/items/:id`）, `CapturePage.vue`（`/capture`）
- データ: `src/lib/db.ts`（Dexie）, `src/lib/useItems.ts`（composable）
- 部品: `src/components/ItemCard.vue`, `LoadingSpinner.vue`
- 既存 `Item`: `{ id, imageData(単一Base64), urgency:0|1|2|3, status:'want'|'hold'|'discarded', blockers[], memoryNote?, createdAt, updatedAt }`

### 本デザインで必要になる主な変更
1. **写真を複数枚に**: `imageData: string` → `photos: string[]`（Base64配列）＋ `coverIndex: number`（表紙）。
2. **品名フィールド追加**: `name: string`（銘々札の主役）。空は「無名」として許容。
3. **基本情報の追加**: `price?: number`（購入価格）, `lastUsedAt?: string`（最後に使った）, `location?: string`（保管場所）。
4. **手放し日**: `releasedAt?: number`（`status` が捨離になった日時。「◯日間そばに」の算出に使用）。
5. **捨てたい度のUI順とラベル**: 表示は **今すぐ / 捨てたい / 迷い / 残す** の4段階。既存 `urgency` 値との対応は §4 の表を参照（値は温存し、表示ラベル・順序・色だけ本デザインに合わせる）。
6. **status の意味付け**: 一覧タブは「候補」と「捨離」の2つ。候補 = まだ手放していない（既存 `want`/`hold` 相当）、捨離 = 手放し済み（既存 `discarded`）。「残す」は候補内の度合いであって別ステータスではない。
7. **保存は自動保存**（手動保存FABは廃止）。退避＝候補への追加が即確定。
8. **ルーティング追加**（§3）: 撮影の判定画面、捨離タブ、編集、全画面ビューア、手放す確認・完了。

> Dexie のスキーマ変更にはマイグレーション（`db.version(2)`）が必要。`imageData` → `photos[0]` へ移行するアップグレード関数を用意すること。

### 円相モチーフのルール（重要）

- **手放し（朱）= 閉じた円相**（`assets/enso-shu-512.png`）。「変容・完結」。閉じて完結し、写真は灰へ薄れる。
- **確認（鈍色）= 円相グレーアウト**。青の円相を `filter:grayscale(1); opacity:.34〜.38` で沈める。弔いの静けさ。
- コピー原則: 「保存」ではなく **「退避」**。手放しは「削除ではなく **変容**」。

## 3. 画面フロー & ナビゲーション

`不断捨離_全体フロー図.dc.html` に俯瞰図あり。矢印は **退避＝青 / 手放し＝朱 / 通常遷移＝鼠**。

```
候補(or 捨離)タブ ──カメラFAB──▶ 撮影 ──▶ 捨てたい度 ──退避アニメ──▶ 候補
                                        └─「書き留める」──▶ 詳細(編集)
候補カード ──▶ 詳細(概要) ⇄ 詳細(編集) / 全画面ビューア
詳細(概要) ──▶ 手放す確認 ──▶ 手放し完了アニメ ──▶ 捨離
捨離カード ──▶ 捨離の詳細 ──「やっぱり候補に戻す」──▶ 候補
候補が空 ──▶ エンプティ表示
```

### ナビゲーションモデル
- **ボトムナビ**: 中央に大きな **カメラFAB**（枠から少し浮かせて上にはみ出す）、左右に **候補 / 捨離** タブ。候補と捨離は同一ボトムナビのタブ切替。
- **← 戻る（プッシュ＝全画面ページ）**: 概要・編集・捨離の詳細（カードから開く）。
- **✕ 閉じる（重なる全画面モーダル）**: 手放す確認・全画面ビューア（アクションから重ねて開く）。
- 詳細系は **全画面ページ**（写真ヘッダー→本文が地続き。ボトムシートではない）。

### 推奨ルート（vue-router）
| ルート | 画面 |
|---|---|
| `/items` | 候補（ワークベンチ）※空はエンプティ |
| `/discarded` | 捨離（タブ）※空はエンプティ |

## 4. 捨てたい度（4段階）の対応表

表示は左から **今すぐ → 捨てたい → 迷い → 残す**。既存 `urgency` 値を温存する場合の対応（要チーム確認）:

| 表示ラベル | 意味 | 色 | 既存 urgency | 退避後の所属 |
|---|---|---|---|---|
| 今すぐ | すぐ手放したい | `#dc2626` | 3 (Now) | 候補（want系） |
| 捨てたい | 手放したい | `#ea580c` | 2 (Want) | 候補（want系） |
| 迷い | 保留（既定） | `#ca8a04` | 1 (Hesitate) | 候補（hold系） |
| 残す | 残す方向 | `#16a34a` | 0 (Keep) | 候補（hold系） |

- 退避時の既定は **「迷い」**（決断ゼロでも進める）。あとから編集で変更可。
- これらは **候補内の度合い**。手放し（捨離へ移動）は別アクション。

## 5. 画面別 詳細仕様（全11画面）

各画面のカンプは `不断捨離_完成カンプ.dc.html` の該当番号を参照。端末サイズは iPhone 相当（375×812 でデザイン）。

### 01. 候補（ワークベンチ）＝ ホーム
- **目的**: 退避したモノの一覧。ここから撮影・詳細閲覧へ。
- **レイアウト**: 上=ヘッダー（アプリ名「不断捨離」30px/700 Shippori Mincho ＋ 右に設定ボタン `settings`。設定項目が未定の間は tap で準備中の案内）。その下に横スクロールの絞り込み: 度合いチップ（全部/今すぐ/迷い…、選択は勝色塗り）。迷った理由（タグ）は記録として保持するが絞り込みチップには使わない。本体= **2列グリッド** のカード（切り替えでコンパクト縦リスト、件数表示つき）。最下部=ボトムナビ。
- **カード（ItemCard）**: 白面 `#fff`・角丸12・境界・影。上部 `aspect-ratio:1/1` の表紙写真。
  - 左上に **枚数バッジ**（複数写真の時だけ）: 黒半透明 `rgba(0,0,0,.55)`＋`copy`系アイコン＋数字。
  - 右上に **度合いバッジ**: 白半透明ピル＋度合い色アイコン＋ラベル。
  - 下部: 品名（Shippori 13/600・省略記号）＋ 左に理由タグ小片、右に相対日時（今日/1日前…）。
- **ボトムナビ**: 高さ82、白面、上境界。左「候補」（勝色・アクティブ）右「捨離」（`#8f887c`）。中央に **カメラFAB**: 64×64 円、勝色、白5px枠、`camera` アイコン白28、`bottom:34px` で浮上。
  - 初回起動時のみカメラ上に吹き出し「捨てるか迷ったら、撮る」（墨、下向き三角）→ スクロール/操作で消える。再訪時は非表示。
- **操作**: カード tap → 概要(05)。FAB tap → 撮影(02)。捨離タブ tap → 捨離(10)。

### 02. 撮影・捨てたい度
- **目的**: 1枚撮る→その場で複数枚追加→捨てたい度を選んで退避。
- **レイアウト**: 上=撮影プレビュー（アプリ内カメラ、FAB から中間タップなしで即起動、`flex:1`）。プレビュー上にズームコントロール（ピンチ＋ `1x`/`2x` プリセット）と前面/背面切替 `switch-camera`、ライブラリ選択 `images`。上左に **「✕ やめる」**、上右に **「↻ 撮り直す」**（黒半透明ピル、`x` / `rotate-ccw`）。プレビュー下端に **写真ロール**: 撮影済みサムネ（表紙は白アウトライン）＋末尾に破線「＋ もう1枚」タイル（カメラ/ライブラリ選択）。
- 下=和紙シート（`#f4f1ea`、上端角丸26）: 「このモノ ・ **2** 枚」→ 見出し「この迷いは、どのくらい？」(Shippori 18/600) → **4択の度合い**（円形アイコン＋ラベル、選択中は勝色＋円相を背後に薄く重ねる。円相は選択肢の枠内に収める）→ 主ボタン **「退避して、次を撮る」**（勝色ピル、`camera`）→ 副ボタン **「退避して、書き留める」**（勝色アウトラインピル、`edit-3`）。退避アクションは写真が1枚以上あるときだけ表示する（写真ゼロではまず撮影へ誘導）。
- **操作**: 「もう1枚」→ 同じアイテムに写真追加（カメラ撮影 or 端末ライブラリから選択）。「次を撮る」→ 退避アニメ(03)→カメラに戻りループ。「書き留める」→ 退避後、編集(06)へ。✕やめる=保存せず一覧へ（初回撮影の✕も候補一覧へ戻る）。↻=この1枚を破棄して再撮影。前面/背面切替時はズームを 1x に戻す。

### 03. 退避（青の円相）＝ 完了演出
- **目的**: 退避した瞬間の一瞬の演出。手を止めずリズムを刻む。
- **見た目**: 和紙背景中央に **青の開いた円相**（`pwa-512x512.png`）、中に写真、右下に緑チェック（`#4a8060` 円＋`check`）。下に「退避しました」(Shippori 22/700)＋「本日 **9** つめ」。画面下部に「— 見届けたら、次へ —」＋ **「次を撮る」**、**「候補一覧へ戻る」**、常時表示の **「退避を取り消す」**（`rotate-ccw`）。
- **アニメ**（§6）: 円相が左下から時計回りに一筆で描かれ、少し開けて止まる（閉じきらない＝思想）。自動復帰はせず、ユーザーが余韻を見届けてから次の導線を選ぶ。取り消しはしばらく常時表示。

### 04. 候補エンプティ
- **目的**: 候補が空のときの受け皿。撮影へ誘導。
- **見た目**: ヘッダーは 01 と同じ。本体中央に薄い円相（`opacity:.28`）を背に、「**捨てるか迷ったら、撮る。**」(Shippori 22/700)＋「迷いは、捨てなくていい。ひとまず、ここへ退避しましょう。」(Shippori 14, `#8f887c`)＋ 下向き矢印＋「下の **カメラ** から」。ボトムナビ＋FABは 01 と同じ。

### 05. 詳細＝概要（銘々札）※全画面ページ（←戻る）
- **目的**: モノのホーム。写真・記録の要約を見る／編集・手放すへ。
- **レイアウト**: 写真ヘッダー（**カルーセル**、高さ250、横スワイプ、`1 / 3` カウンタ＋ドット、右下「写真を足す」`plus`）。**その下に本文が地続き**（角丸シートの浮きなし＝フルページ）。中央寄せで品名(Shippori 24/700)＋メタ（度合い色ドット＋「迷い ・ 写真3枚 ・ 3月12日」）。円相の小さな区切り線。**記録カード**（`#fbfaf6`・角丸16）: 「記録」見出し＋右上「編集」リンク、購入価格・保管場所などの行、区切り、想いメモの抜粋(Shippori)。最下に主ボタン **「手放す」**（墨・角丸14、`trash-2`）。
- 上左に ← 戻る（半透明丸ボタン）、上右に削除（朱）※任意。
- **操作**: 「編集」→ 06。写真 tap → 全画面ビューア(07)。「写真を足す」→ カメラ撮影 or 端末ライブラリから選択。品名 tap → その場編集（遷移なしで保存）。「手放す」→ 手放す確認(08)。← → 一覧。

### 06. 詳細＝編集（記録）※全画面ページ
- **目的**: 全項目を入力する記録専用画面。
- **レイアウト**: 上=ヘッダーバー（「キャンセル」／中央「記録する」Shippori 16／「完了」勝色700、下境界）。本体（縦積み・gap18）:
  - **写真ロール**: ラベル「写真（3枚）」＋「長押しで並べ替え」。88×88サムネ横並び、各サムネ右上に削除（`x`）、先頭に「表紙」バッジ（勝色）。末尾に破線「＋ 足す」タイル。
  - **品名**: 入力（白・角丸12、Shippori 17）。
  - **捨てたい度**: 4分割ピル（選択=`#a8892e` 塗り）。
  - **基本情報**: 購入価格（`yen`系）/最後に使った（`calendar`、カレンダー/デートピッカーで選択＋「今日」クイック入力）/保管場所（`map-pin`）の行リスト（白・角丸12・区切り）。未入力はプレースホルダ灰。写真の「＋ 足す」はカメラ/ライブラリ選択。
  - **モノへの想い・メモ**: 複数行テキスト（`#fbfaf6`・角丸12、Shippori 14/1.8）。
- **保存**: 「完了」で保存（自動保存前提）→ 概要(05) に着地。

### 07. 全画面ビューア ※モーダル（✕）
- **目的**: 写真をスワイプ閲覧。
- **見た目**: 背景 `#0c0c0e`。上=✕（左）／「2 / 3」（中）／`more-horizontal`（右）。3点メニューは表紙設定・削除、手放し済みはさらに「画像をライブラリに保存」。中央=大きな写真、左右に隣写真がのぞく。下=サムネ帯（現在に白アウトライン）。左右スワイプで移動、サムネ tap でジャンプ。

### 08. 手放す確認 ※モーダル（✕）＝ 鈍色・静けさ
- **目的**: 「手放す＝変容」への静かな決心（まだ手放していない）。confirm ダイアログは使わず全画面の儀式。
- **見た目**: 画面 `#eeecea`。上左に ✕。中央に **円相グレーアウト**（青の円相を `grayscale(1); opacity:.38`）＋中に写真（暗）。見出し「祖母のマグカップ を手放しますか？」(Shippori 22/700)＋本文「そっと、記憶へ移します。写真と記録は「捨離リスト」に残る。これは削除ではなく、**変容**です。」(Shippori 14, `#6b6459`)。下部に主ボタン **「手放す（捨離リストへ）」**（**鈍色 `#57565b`** ピル、`trash-2`）＋「まだ、そばに置く」（テキストボタン）。
- **操作**: 「手放す」→ 完了演出(09)。✕/「まだ」→ 概要へ戻る。

### 09. 手放し完了（朱の円相）＝ 完了演出
- **目的**: 手放しの完結を前向きに報う。
- **見た目**: 画面 `#efece3`。中央に **朱の閉じた円相**（`enso-shu-512.png`）＋中の写真が灰へ薄れる。「円が、ひとつ結ばれました」(Shippori 22/700 朱)＋「〜を手放しました。記録は「捨離」に残ります。」(Shippori 14, `#7a5348`)。下に「— 捨離リストへ移りました —」＋ **「捨離リストを見る」**＋ **「候補一覧へ戻る」**（退避完了(03)と対称の導線）。
- **アニメ**（§6）: 朱の円相が一筆で **閉じて完結**（下の隙間まで結ぶ）、同時に写真 grayscale フェード、テキストが遅延で立ち上がる。自動遷移はせず、ユーザーが余韻を見届けてから `捨離リストを見る` で捨離(10)へ、または `候補一覧へ戻る` で候補(01)へ移る。`releasedAt = Date.now()` を記録。

### 10. 捨離（タブ）※ボトムナビの捨離
- **目的**: 手放し済みの一覧を静かに振り返る。
- **見た目**: ヘッダー（同）。中央に「手放した記録」の区切り（Shippori・`trash-2`）。**候補と同じ2列カード**（切り替えでコンパクト縦リスト、件数表示つき）だが写真は `grayscale(.5); opacity:.9`、右下に **「手放し済み」** 帯（墨ピル）。品名＋相対日時。ボトムナビは「捨離」アクティブ。カード tap → 捨離詳細(11)。
- 捨離が空のエンプティも用意（候補エンプティに準じたトーン）。

### 11. 捨離の詳細 ※全画面ページ（←戻る）
- **目的**: 手放した記録を振り返る（品名を除き読み取り専用。品名のみ tap でその場編集でき、`status`/`releasedAt` は変わらない）。取り消せる安心。
- **レイアウト**: 写真ヒーロー（暗）＋右上に表示中の画像をライブラリに保存する `download`、右下に **朱の閉じた円相バッジ＋「手放し済み」**。**下に本文が地続き**（フルページ）。品名(Shippori 24/700)＋「3月28日に手放しました ・ 12日間そばに」（朱、`releasedAt` と `createdAt` から算出）。朱の円相区切り。**記録カード（読み取り専用）**: 購入価格・迷った理由＋想いメモ（Shippori）。下に **「やっぱり候補に戻す」**（勝色アウトライン、`rotate-ccw`）＋注記「手放したことにしたけれど、まだ手元にある時に」。
- **操作**: 画像の `download` または写真 tap → 全画面ビューアの3点メニューから、表示中の1枚を端末ライブラリに保存。「候補に戻す」→ `status` を候補へ、`releasedAt` クリア → 候補(01)。

## 6. インタラクション & アニメーション

キーフレーム名はカンプ内 `<style>` を参照（`ensoDraw` / `ensoPop` / `ensoCheck` / `ensoFade` / `photoFade` / `riseFade`）。

### 退避（青）— 撮影ループ内
1. **0〜0.8s**: 写真が中央に収まり、青の円相が左下から時計回りに一筆で描かれる（SVGマスク＋`stroke-dashoffset` アニメ、**閉じきらず少し開ける**）。
2. **0.85s**: 緑チェックがふわりと出現（`ensoCheck`）、「退避しました／本日◯つめ」。
3. **1.2s〜**: 画面に留まり、「次を撮る」「候補一覧へ戻る」「退避を取り消す」を表示する（取り消すと直前の退避を無効化）。

### 手放し（朱）— 儀式の完了
1. **0〜0.5s**: 朱の円相が一筆で描かれ、下の隙間まで **閉じて完結**。同時に写真が `photoFade`（1s）で灰へ。
2. **0.7s〜**: 「円が、ひとつ結ばれました／手放しました」（`riseFade`）。
3. **1.1s〜**: 画面に留まり、「捨離リストを見る」から捨離リストへ移動する（記録は残る＝変容）。

### 遷移
- 概要/編集/捨離詳細 = **プッシュ（右から）**、← で戻る。
- 手放す確認/ビューア = **モーダル（下または不透明オーバーレイ）**、✕ で閉じる。
- カルーセル/ビューアは横スワイプ、写真ロールは長押し並べ替え。

### 状態
- **ローディング**: 既存 `LoadingSpinner.vue` を流用。
- **エンプティ**: 候補・捨離それぞれに専用エンプティ（§04, §10）。
- **自動保存**: 入力は都度保存。明示的な保存ボタンは持たない（編集の「完了」は画面を閉じるだけ）。

## 7. 状態管理・データ

- 永続化は既存どおり **Dexie**（`src/lib/db.ts`）。`useItems.ts` の composable を拡張。
- 追加/変更フィールド（§1）を反映した `Item` 型へ更新し、`db.version(2)` でマイグレーション（`imageData` → `photos:[imageData]`, `coverIndex:0`, `name:''` 補完）。
- 一覧は `liveQuery` で購読（既存パターン）。候補 = 未手放し、捨離 = `releasedAt` 有り（または `status==='discarded'`）で振り分け。
- 「◯日間そばに」= `Math.round((releasedAt - createdAt)/86400000)`。相対日時（今日/◯日前）表示のユーティリティを用意。
- 退避の取り消し・「候補に戻す」= status/releasedAt の更新のみ（物理削除しない）。

## 10. 世界観（実装時の判断基準）
- **「決断しないで、退避する」**。円は閉じきらなくていい＝優柔不断の肯定。
- コピーは「保存」ではなく「退避」、手放しは「削除ではなく変容」。
- 派手なアラート赤は使わない。手放しは朱（浄め・変容）、確認は鈍色（弔いの静けさ）。
- 静けさ・余白・和紙の温かみを最優先。装飾より「間」。
